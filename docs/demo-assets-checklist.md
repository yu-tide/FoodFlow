# Demo Assets Checklist — 需要手动补充的截图/GIF

以下截图需要在实际运行环境中手动截取（建议使用本地 Mock 模式运行，无需部署到服务器）。

## 必须（README 用）

- [ ] **仪表盘页面** (`/dashboard`)
  - 要求：展示完整仪表盘 — 当日热量环形图、三大宏量营养素进度条、近期餐食列表、周热量趋势图
  - 建议尺寸：1200×800

- [ ] **上传页面** (`/upload`)
  - 要求：展示图片上传界面，含餐食类型选择器和备注输入框
  - 建议尺寸：800×600

- [ ] **AI 分析进度** (`/analyze/[taskId]`)
  - 要求：展示 7 阶段进度条在中间阶段的界面（如 AI_RECOGNITION 阶段）
  - 建议尺寸：800×600

- [ ] **食物记录详情/确认页面** (`/records/[id]` 或 `/confirm/[recordId]`)
  - 要求：展示 AI 分析结果 — 食物名称列表、各成分克数、营养汇总、AI 总结文本
  - 建议尺寸：800×600

- [ ] **AI 助手对话** (浮窗面板)
  - 要求：展示一次完整的对话交互，如用户问「我今天还能吃什么」的问答
  - 建议尺寸：400×600（面板特写）

- [ ] **设置页面** (`/settings`)
  - 要求：展示身体信息 + 营养目标设置区域
  - 建议尺寸：800×600

## 建议（增强展示效果）

- [ ] **完整流程 GIF**：上传图片 → 分析进度条 → 结果展示 → 用户确认 → 仪表盘更新
  - 要求：流畅的 15-30 秒动画
  - 工具：ScreenToGif (Windows) / Kap (macOS) / OBS

- [ ] **每周统计页面** (`/statistics/weekly`)
  - 要求：展示日均热量柱状图、宏量营养素分布、蛋白达标天数
  - 建议尺寸：800×600

- [ ] **用户设置 — BMR 计算器** (`/settings`)
  - 要求：展示点击「推荐目标」后的 BMR/TDEE 计算结果
  - 建议尺寸：800×600

- [ ] **记录列表页** (`/records`)
  - 要求：展示按日期筛选的记录列表
  - 建议尺寸：800×600
